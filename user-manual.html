How the Purchase Flow Works

Buyers do not need to create an account. They provide email + phone during checkout, pay, and immediately get access. Access is tied to an httpOnly browser cookie.

→
Visitor browses courses on the home page
→
Clicks a course → sees description, curriculum, price, preview videos
→
Clicks “Buy Now — ₹999”
→
Guest checkout modal opens → enters email + phone
→
Clicks “Continue to Payment” → Razorpay checkout opens (UPI / Card / Netbanking)
→
Payment completes → signature verified → purchase marked completed
→
httpOnly cookie set → redirected to “Payment Successful!” page
→
Clicks “Start Watching” → full course player with all lessons — lifetime access

Trade-off: If a buyer clears cookies, switches browsers, or uses a new device, they lose access. This is solved by the Recover Access flow.

How Content Protection Works

Premium course videos are YouTube unlisted links. Anyone with the link can watch. The site prevents non-paying users from ever seeing these links through 2 layers of protection:

Layer 1
Cookie Validation

When someone visits /courses/guitar-basics/watch, the server reads the purchase_access_{courseId} cookie. If missing → redirected to the course detail page. They never reach the player.

Layer 2
Server-Side Database Verification

Even if a cookie exists, the server validates its value (a UUID access_token) against the purchases table, checking for a completed purchase with that token for that course. A guessed or forged cookie will fail this check. YouTube URLs are only loaded if verification passes.

What Each User Sees

User State Course Detail Page Course Player
No purchase cookie Curriculum titles + lock icons, “Buy Now” button, preview videos Redirected to course detail
Valid purchase cookie Curriculum + play icons, “Continue Watching” Full video player with all lessons

How Razorpay Protects Payments

Razorpay handles the money. Here's the exact flow:

1
User clicks “Buy Now” → enters email + phone → server creates a Razorpay order
2
Razorpay checkout modal opens → user pays via UPI / Card / Netbanking
3
Razorpay returns a payment ID + cryptographic signature
4
Server verifies the signature using HMAC-SHA256 with your secret key
5
Signature valid → purchase marked completed → httpOnly cookie set
6
User can now access all course videos — lifetime access

Why this is secure: Only Razorpay can produce a valid signature. Only your server has the secret key to verify it. No one can fake a payment — the purchase is only marked “completed” after cryptographic verification passes.

How to Recover Access

If a buyer clears their browser cookies, switches to a new browser, or uses a different device, they can restore access without contacting support:

1
Visit /recover-access (or click “Already purchased? Recover access” on any course page)
2
Enter the email used during purchase
3
Receive a magic link via email (expires in 30 minutes)
4
Click the link → all purchase cookies are restored on the current browser
5
See the Access Restored! confirmation page

How it works under the hood: The server generates a signed HMAC token encoding email + timestamp (no database changes needed). When the link is clicked, it verifies the token’s signature, checks it hasn’t expired, fetches all completed purchases for that email, and sets a cookie for each course.

How to Upload a Paid Course

The site does not host any video files. All videos live on YouTube as unlisted uploads. You paste URLs into the admin panel.

Step 1: Record & Upload to YouTube

  1. Record your course lessons
  2. Upload each lesson to your YouTube channel
  3. Set each video to “Unlisted” — YouTube Studio → Content → video → Visibility → Unlisted
  4. Copy the URL of each video (e.g., https://www.youtube.com/watch?v=abc123)

Step 2: Create the Course

  1. Go to /admin → Courses → New Course
  2. Fill in: Title, Slug (auto-generated), Description, Price (in Rupees)
  3. Optionally add a Thumbnail URL
  4. Leave Published unchecked while adding videos
  5. Click Create Course

Step 3: Add Videos

  1. From Courses list, click the video icon next to your course
  2. Click Add Video — a form appears with a live YouTube preview
  3. Paste the YouTube URL — the video loads immediately so you can verify it’s correct
  4. Fill in title, duration (optional), and check Free Preview if it’s a teaser
  5. Repeat for all lessons

Step 4: Publish

  1. Edit the course → check Published → Update Course
  2. The course now appears on the home page with the “Buy Now” button

Step 5: Monitor Payments

  • Go to /admin → Payments to see all transactions
  • Each row shows: date, buyer email, course, amount, status, and Razorpay payment ID
  • Cross-reference with your Razorpay dashboard for refunds or disputes

Adding More Lessons Later

Upload the new lesson to YouTube (unlisted) → go to Admin → Courses → video icon → Add Video. Buyers who already purchased see the new lesson immediately.

Replacing a Video

Upload new version to YouTube (unlisted) → delete old entry in Admin → add new one with the new URL.

Admin Video Preview

The Course Videos admin page includes:

Live Preview on Add

When you paste a YouTube URL in the add form, the video loads immediately. You can watch it right there to confirm it’s the right one before saving.

Inline Preview on Browse

Click the expand arrow on any video in the list to see the full embedded player. Verify what your students will see without leaving the admin panel.

Setup Checklist

  1. Create a Supabase project → copy URL + anon key + service role key
  2. Run supabase-migration.sql in Supabase SQL Editor
  3. Create a Razorpay account → copy test key ID + secret
  4. Set up a Gmail App Password for sending recovery emails (Google Account → Security → 2FA → App Passwords)
  5. Update .env.local with all credentials: 
    NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
NEXT_PUBLIC_RAZORPAY_KEY_ID=...
RAZORPAY_KEY_SECRET=...
GMAIL_USER=...
GMAIL_APP_PASSWORD=...
NEXT_PUBLIC_APP_URL=http://localhost:3000
  6. Run npm run dev → open http://localhost:3000
  7. Register an admin account → set as admin in Supabase SQL: 
    UPDATE public.profiles SET role = 'admin' WHERE email = 'your@email.com';
  8. Go to /admin → start adding courses and videos
  9. For production: deploy to Vercel, switch Razorpay to live mode, update NEXT_PUBLIC_APP_URL
